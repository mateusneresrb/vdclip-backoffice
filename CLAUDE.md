# VDClip Dashboard

SPA dashboard for VDClip — video clip management platform.

## Tech Stack

- **Runtime**: Bun | **Build**: Vite 7 | **Framework**: React 19 + TypeScript (strict)
- **Styling**: Tailwind CSS 4 + shadcn/ui (New York) | **Icons**: lucide-react
- **Routing**: TanStack Router (file-based) | **Data**: TanStack Query | **State**: Zustand (persist)
- **i18n**: react-i18next — EN, PT-BR, ES | **Forms**: react-hook-form + zod
- **Lint**: @antfu/eslint-config + Prettier (no semicolons, single quotes, trailing commas)

## Commands

```bash
bun dev          # Dev server + generates routeTree.gen.ts
bun run build    # tsc + vite build
bun run lint     # ESLint
bun run lint:fix # ESLint --fix
bun run format   # Prettier
```

## Architecture

```
src/
  components/layout/  → sidebar, header, theme, language switcher
  components/ui/      → shadcn (NEVER edit manually, NEVER place custom code here)
  features/{name}/    → components/, hooks/, types/, index.ts (self-contained modules)
  routes/             → TanStack Router file-based (auto-generates routeTree.gen.ts)
  routes/_authenticated/ → protected pages (sidebar layout)
  stores/             → Zustand stores
  providers/          → React context providers
  hooks/              → shared hooks
  lib/                → utils.ts
  i18n.ts             → i18next config
  router.tsx          → router instance + type registration
```

## Code Style

- Named exports only — no `export default`
- Import order: external libs → `@/` aliases → relative imports
- Path alias: `@/` → `src/`
- Static data outside components with `as const`
- Use `cn()` from `@/lib/utils` for conditional class merging
- Prefer composition (children/slots) over configuration (many props)

## Critical Rules (DON'T)

- **DON'T** edit `src/components/ui/` — use `bunx shadcn@latest add <name>` to add/update
- **DON'T** edit `src/routeTree.gen.ts` — auto-generated by TanStack Router plugin
- **DON'T** hardcode text strings — ALL user-facing text uses `t('key')` with react-i18next
- **DON'T** add translations to only 1 language — ALWAYS update EN + PT-BR + ES
- **DON'T** use `bg-white`, `text-black`, `bg-gray-*` — use semantic tokens (`bg-background`, `text-foreground`)
- **DON'T** use `min-h-screen` — use `min-h-svh` (accounts for mobile browser chrome)
- **DON'T** use `export default` — named exports only
- **DON'T** use `<a>` for internal navigation — use `<Link>` from `@tanstack/react-router`
- **DON'T** store auth tokens in localStorage — Zustand persist is for UI preferences only

## Gotchas

- TanStack Router types only work after `routeTree.gen.ts` exists — run `bun dev` once after creating routes
- shadcn Sidebar has its own `SidebarProvider` context — don't confuse with Zustand sidebar store
- i18n uses HTTP backend loading from `public/locales/` — translations are NOT bundled, loaded at runtime
- Tailwind CSS 4 uses `@import "tailwindcss"` NOT `@tailwind` directives
- Theme uses CSS class strategy (`.dark` on `<html>`) NOT `prefers-color-scheme` media query

## Adding a New Route

1. Create `src/routes/_authenticated/{name}.tsx` with `createFileRoute`
2. Add icon + entry to `navItems`/`footerItems` in `app-sidebar.tsx`
3. Add label to `routeLabels` map in `header.tsx`
4. Add `nav.{name}` to ALL 3 locale files in `public/locales/`
5. Run `bun dev` to regenerate route tree

## Adding a New Feature

1. Create `src/features/{name}/` with `components/`, `hooks/`, `types/`, `index.ts`
2. Query key factory in hooks: `featureKeys.all`, `.list(filters)`, `.detail(id)`
3. Create the route, nav item, and translations (see above)
4. If feature has many strings, create a dedicated namespace: `public/locales/{lng}/{name}.json` + register in `i18n.ts` `ns` array

## Path-Specific Rules

Detailed rules for components, routes, translations, styles, and state are in `.claude/rules/`. These load automatically when working on matching file paths.
